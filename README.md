## 📒 notes.md Template

Every problem folder should have a `notes.md` like this:

```markdown
## Problem
[Link to problem]

## Approach
Describe your thinking in 2–3 lines. What pattern did you use?
(e.g. "Used a HashMap to store seen values and check for complement in O(1) time.")

## Complexity
- **Time:** O(n)
- **Space:** O(n)

## What tripped me up
Write anything that confused you or took time to figure out.
This is the most useful section for interview prep.

## Tags
`HashMap` `Arrays` `Easy`
```

## 🔁 Weekly Review Habit

Every Sunday, pick 2–3 problems you solved that week and **re-solve them without looking**.
If you can't, add them to a `revisit/` folder.

## 🛠️ Languages

Solutions are written in **C** and **C++**.

| File | Language |
|------|----------|
| `solution.c` | C (for pointer/memory practice) |
| `solution.cpp` | C++ (for STL — vectors, maps, sets, etc.) |

Use C when the problem is about raw arrays, pointers, or manual memory. Use C++ when you need STL containers or OOP patterns.
